1 Overview

1.1 About this API

This documentation describes the resources that compose the official Welcu API v1.

This API is currently under development so this document may change as the API evolves. This document source is available at it's GitHub repository and we encourage developers to point out any problem with the API documentation or behavior, as well as any feature suggestion to our issue tracker.

Undocumented data delivered through the API responses should be treated as experimental and their structure might change.

1.2 Authentication

This API requires an API token per event. This token can be obtained from the integrations menu of your organization.

Every method of this API require an event token which must be passed in the HTTP request Authorization header like: 

Authorization: event HERE_GOES_THE_TOKEN

So for example using curl. Every request must be called like:

curl -i -H 'Authorization: event HERE_GOES_THE_TOKEN' \
  https://api.welcu.com/v1/api_method.json

1.3 Errors

Errors are delivered via HTTP Status Codes along with a JSON payload with the description of the error
401 Unauthorized

This code is returned when the event token is wrong/disabled/empty 

curl -i https://api.welcu.com/v1/event.json
Response
TODO

404 Not Found

This code is returned when the API can't find the resource you are querying.

406 Not Acceptable

This code is returned when the change you are trying to perform is not possible

422 Unprocessable Entity

This error is returned when there are errors on the parameters given to the API

3 Attendees

3.1 Get attendee list

GET
/event/attendees[/:state].json
Using this endpoint you can get the list of attendees along with it's tickets
Available parameters
Parameter Description
state (optional) To filter the list by the attendee state. The available states are:
  • checked Filter only attendees with checked tickets
  • unchecked Filter only attendees with unchecked tickets
ticket_id (optional) To filter the attendees with tickets for a given ticket_id
since (optional) An ISO8601 encoded date to filter attendees updated since the given date. 
curl -i -H 'Authorization: event TOKEN' \
  https://api.welcu.com/v1/event/attendees/checked.json

3.3 Check an attendee's ticket

POST
/event/attendees/code/:code/check.json
This endpoint allows you to mark the ticket as checked
Available parameters
Parameter Description
at (optional) An ISO8601 encoded date to specify at which time the ticket was checked. The current time is used if left blank. 
curl -i -H 'Authorization: event TOKEN' \
  -d 'at=2012-11-16T08:27:05-03:00' \
  https://api.welcu.com/v1/event/attendees/code/sc1f5e6/check.json
Response
HTTP/1.1 202 Accepted
Content-Type: application/json; charset=utf-8
Cache-Control: no-cache
Connection: close
